What is an Exception?

An exception is an event that disrupts the normal flow of a program. Java uses a hierarchy rooted at Throwable, branching into Error (JVM-level) and Exception (application-level).

Checked vs. Unchecked

  • Checked — must be declared or caught (e.g. IOException, SQLException).
  • Unchecked — extends RuntimeException; no obligation to catch (e.g. NullPointerException, ArrayIndexOutOfBoundsException).

try-catch-finally

Java 
try {
    FileReader fr = new FileReader("data.txt");
} catch (FileNotFoundException e) {
    System.err.println("File not found: " + e.getMessage());
} finally {
    // always runs — release resources here
}

try-with-resources (Java 7+)

Java 
try (BufferedReader br = new BufferedReader(new FileReader("data.txt"))) {
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();
}
// br.close() is called automatically

Custom Exceptions

Java 
public class PaymentException extends RuntimeException {
    private final int errorCode;

    public PaymentException(String msg, int errorCode) {
        super(msg);
        this.errorCode = errorCode;
    }
    public int getErrorCode() { return errorCode; }
}

Best practice: Prefer specific exception types over generic Exception — it improves readability and lets callers handle only the errors they care about.

Creating Threads

Two classic approaches: extending Thread or implementing Runnable. Modern code should use ExecutorService instead of raw threads.

Java 
// Preferred: Runnable via ExecutorService
ExecutorService pool = Executors.newFixedThreadPool(4);
pool.submit(() -> {
    System.out.println("Running in: " + Thread.currentThread().getName());
});
pool.shutdown();

Synchronization

Use synchronized to protect shared mutable state from concurrent access:

Java 
public synchronized void increment() {
    counter++;  // thread-safe
}

// Or use AtomicInteger for simple counters
AtomicInteger counter = new AtomicInteger(0);
counter.incrementAndGet();

CompletableFuture (Java 8+)

Java 
CompletableFuture.supplyAsync(() -> fetchData())
    .thenApply(data -> transform(data))
    .thenAccept(result -> save(result))
    .exceptionally(ex -> { log(ex); return null; });

Avoid sharing mutable state across threads without proper synchronization — it leads to race conditions that are notoriously hard to reproduce and debug.

Legacy IO vs NIO.2

The legacy java.io.File API has many limitations. Java 7 introduced java.nio.file with a far richer and more reliable API.

Reading a File

Java 
Path path = Paths.get("data/config.txt");

// Read all lines
List<String> lines = Files.readAllLines(path, StandardCharsets.UTF_8);

// Stream lines lazily (efficient for large files)
try (Stream<String> stream = Files.lines(path)) {
    stream.filter(l -> l.startsWith("#")).forEach(System.out::println);
}

Writing a File

Java 
Files.writeString(path, "Hello World\n", StandardOpenOption.CREATE);

// Append mode
Files.writeString(path, "More content\n",
    StandardOpenOption.APPEND, StandardOpenOption.CREATE);

Working with Paths

Java 
Path dir = Path.of("/app/data");
Files.createDirectories(dir);
Files.copy(src, dest, StandardCopyOption.REPLACE_EXISTING);
Files.move(src, dest);
Files.delete(path);

// Walk directory tree
Files.walk(dir)
     .filter(Files::isRegularFile)
     .forEach(System.out::println);

Always use Files.lines() inside a try-with-resources block — the underlying stream holds an open file handle that must be closed.

Stream Pipeline

A stream pipeline has a source, zero or more intermediate operations, and a terminal operation. Intermediate operations are lazy — they execute only when a terminal operation is invoked.

Java 
List<String> seniors = employees.stream()
    .filter(e -> e.getYearsExp() > 10)
    .map(Employee::getName)
    .sorted()
    .collect(Collectors.toList());

Collectors

Java 
// Group by department
Map<String, List<Employee>> byDept =
    employees.stream()
             .collect(Collectors.groupingBy(Employee::getDept));

// Average salary
double avg = employees.stream()
    .collect(Collectors.averagingDouble(Employee::getSalary));

Choosing the Right Collection

  • ArrayList — fast random access, slow insertion at middle.
  • LinkedList — fast insertions/deletions, slow random access.
  • HashMap — O(1) average get/put; not thread-safe.
  • ConcurrentHashMap — thread-safe alternative to HashMap.
  • TreeMap — sorted keys, O(log n) operations.
  • LinkedHashMap — maintains insertion order.

For concurrent environments, prefer ConcurrentHashMap or Collections.synchronizedMap() over plain HashMap.

☕ Java › JVM Internals
JVM Internals
HeapGCClassLoaderJIT
Advanced

Memory Areas

  • Heap — object storage, divided into Young Gen and Old Gen.
  • Stack — per-thread, stores frames (local vars + operand stack).
  • Metaspace — class metadata (replaced PermGen since Java 8).

Garbage Collection

Modern JVMs ship with multiple GC algorithms: G1 (default since Java 9), ZGC (low-latency), Shenandoah. Tune with -Xms, -Xmx, and GC-specific flags.

Enable GC logging in production (-Xlog:gc*) to spot memory leaks and long stop-the-world pauses before they become incidents.

Security Filter Chain

Spring Security intercepts every request through a chain of filters. Define your security rules via SecurityFilterChain bean.

public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    return http
        .csrf().disable()
        .authorizeHttpRequests(auth -> auth

🔗 Microservices › Service Communication
Service Communication

Architecture

Sync vs. Async

Synchronous (REST/gRPC) — simple request-response; tight coupling. Use when an immediate response is required.

Asynchronous (Kafka/RabbitMQ) — decouples producer and consumer; improves resilience and scalability. Use for event-driven workflows.

At JP Morgan Chase, payment events are published to Kafka topics, allowing downstream services (notifications, audit) to react independently — zero coupling.

🏗️ System Design › Scalability
Scalability Principles
Horizontal ScalingLoad BalancingStateless
Architecture

Horizontal vs. Vertical Scaling

Vertical (scale-up) — add more CPU/RAM to a single node. Has a ceiling and a single point of failure.

Horizontal (scale-out) — add more nodes behind a load balancer. Preferred for modern cloud-native apps.

Stateless Services

Keep services stateless — move session data to a shared store (Redis, Hazelcast). Any node can handle any request, enabling seamless horizontal scaling.

🏗️ System Design › Caching
Caching Strategies
RedisCache-AsideWrite-ThroughTTL
Architecture

Common Strategies

  • Cache-aside (lazy loading) — app checks cache; on miss, loads from DB and populates cache.
  • Write-through — write goes to cache and DB simultaneously; cache always consistent.
  • Write-behind — write to cache immediately, persist to DB asynchronously; risk of data loss.

Always set a TTL on cache entries to prevent stale data and avoid unbounded cache growth.

🏗️ System Design › Database Design
Database Design
IndexingShardingACIDCAP Theorem
Architecture

Indexing Strategy

Create indexes on columns used in WHERE, JOIN, and ORDER BY clauses. Avoid over-indexing — each index slows down writes.

CAP Theorem

Distributed systems can guarantee at most two of: Consistency, Availability, Partition tolerance. Most cloud databases make this a configurable trade-off.

Minimal Dockerfile (Multi-Stage)

Dockerfile 
# Build stage
FROM eclipse-temurin:21-jdk AS build
WORKDIR /app
COPY . .
RUN ./mvnw package -DskipTests

# Runtime stage
FROM eclipse-temurin:21-jre
WORKDIR /app
COPY --from=build /app/target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]

Multi-stage builds keep final images lean — the JDK (build tools) never ends up in the production image.
